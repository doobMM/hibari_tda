🎵 1. 핵심 아이디어

기존 AI 작곡은 대량의 곡을 학습해서 비슷한 곡을 만듭니다 (블랙박스).

이 연구는 다릅니다: 단 한 곡의 수학적 구조를 분석하고, 그 구조를 보존하면서 새로운 곡을 만듭니다.

🎛 2. 음악을 네트워크로 바꾸기

Note = (pitch, duration)

음악의 기본 단위는 음 높이와 음 길이의 쌍입니다. hibari에는 23종의 고유한 note가 있습니다.

예시: 도(C4)를 1박 동안 연주 = (C4, 1), 레(D4)를 2박 = (D4, 2)

이 각각이 네트워크의 노드(node)가 됩니다.

가중치 행렬 = 노드 간 연결 강도

두 note가 곡에서 연달아 연주되는 횟수가 가중치입니다.

거리 행렬

가중치의 역수가 거리입니다. 자주 연달아 나오는 note끼리는 가깝고, 한 번도 연달아 안 나오면 매우 멀리 떨어져 있습니다.

🔎 3. Persistent Homology로 순환 구조 찾기

Persistent Homology는 데이터의 "구멍"을 찾는 수학 도구입니다.

note들을 점으로 보고, 거리가 가까운 것부터 연결하면 — 어느 순간 "루프(cycle)"가 만들어집니다.

직관적 설명: 물이 차오르는 비유

용어 정리

용어의미음악적 해석
Filtration거리 임계값 r을 0에서 점점 키우는 과정연결 기준을 점점 느슨하게
Cycle (H1)루프/고리 구조. 3개 이상의 note가 순환반복 선율, 화성 패턴
Birthcycle이 처음 나타나는 r 값구조가 형성되는 시점
Deathcycle이 삼각형으로 채워져 사라지는 r 값구조가 소멸하는 시점
Barcode각 cycle의 [birth, death] 구간 목록곡의 위상적 지문
Betti Curve각 r에서 살아있는 cycle 수의 그래프곡의 구조적 복잡도 변화

hibari를 분석하면 48개의 고유한 cycle이 발견됩니다. 이것이 곡의 "위상적 뼈대"입니다.

📈 4. 중첩행렬 (Overlap Matrix)

중첩행렬은 "언제 어떤 cycle이 활성화되어 있는가"를 나타내는 이진 행렬입니다.

크기: 1088 시점 x 48 cycles

이 행렬이 음악 생성의 seed(씨앗)입니다. 각 시점에서 어떤 cycle이 활성화되어 있으면, 그 cycle에 속한 note 중에서 연주할 음을 고릅니다.

🎯 5. 위상 구조 보존도와 Cycle 선택

48개 cycle 전부를 사용할 필요는 없습니다. 어떤 subset이 원곡의 구조를 가장 잘 보존하는가?
