🎵 1. 핵심 아이디어

기존 AI 작곡은 수천 곡을 학습하여 비슷한 음악을 만듭니다 (블랙박스).

이 연구는 단 한 곡의 수학적 구조를 분석하고, 그 구조를 보존하면서 새 곡을 만듭니다.

기존 AI 작곡

모차르트 1000곡 학습 → "모차르트 스타일" 생성

왜 그런 곡이 나왔는지 설명 불가

이 연구

"hibari" 한 곡의 뼈대(위상 구조) 추출 → 같은 뼈대 위에 새 음악

작곡 원리를 설명할 수 있음

🎛 2. 음악을 네트워크로 바꾸기

음악의 기본 단위 = 음높이 + 길이. hibari에는 23종의 고유한 음이 있습니다.

두 음이 곡에서 연달아 나온 횟수가 연결 강도(가중치). 그 역수가 거리입니다.

🔎 3. Persistent Homology로 순환 구조 찾기

Persistent Homology는 데이터에서 "고리(루프)" 구조를 찾는 수학 도구입니다.

음들을 점으로 놓고, 가까운 것부터 연결하면 — 어느 순간 루프(cycle)가 형성됩니다.

📈 4. 중첩행렬 — 음악의 뼈대

중첩행렬은 "시간이 흐르면서 어떤 반복 패턴이 켜져 있는가"를 나타내는 이진 행렬입니다.

크기: 1088 시점 × cycle 수

이것이 음악 생성의 seed(씨앗)입니다.

🎯 5. 위상 구조 보존도와 Cycle 선택

발견된 cycle 전부를 쓸 필요는 없습니다. Greedy Forward Selection으로 가장 중요한 cycle부터 하나씩 추가하며, 3가지 지표의 가중 평균으로 보존도를 측정합니다.

🚫 6. 고아 Note 문제와 해결

문제: note 5, 8, 22는 어떤 cycle에도 속하지 않습니다. 48개 전체를 써도 커버 불가 — 구조적 한계.

이 음들은 화음의 일부이지만, 다른 음과의 거리가 상대적으로 멀어서 cycle에 포함되지 못합니다.

해결: 고아 note가 속한 화음이 활성화되는 시점에, 30% 확률로 sampling pool에 주입합니다.

결과: pitch 분포 KL divergence 0.323 → 0.250 (23% 개선)

🎼 7. 음악적 거리 함수

기존: 거리 = 1/(연달아 등장한 횟수). 순수 통계적이라 화성/선율 관계를 무시합니다.

선행연구(정재훈 외, Tymoczko, Catanzaro)를 참고하여 3가지 음악 이론 기반 거리를 추가했습니다.

거리 함수원리비유
Tonnetz (어울림)장3도·완전5도 관계 격자 위 최단 거리"도와 솔"처럼 함께 울리면 어울리는 음이 가까움
Voice-leading (높낮이)반음 수 차이피아노 건반에서 가까이 있는 음이 가까움
DFT (파동)Fourier 변환 후 주파수 공간 L2 거리파동 패턴이 비슷한 음이 가까움

빈도 거리와 음악적 거리를 alpha 비율로 혼합합니다:

hybrid = alpha × 빈도 + (1-alpha) × 음악적

거리 함수 × DL 모델 전체 비교 (JS Divergence, 낮을수록 원곡과 유사)

LSTM은 모든 metric에서 JS > 0.25 — 데이터 1088 시점이 시퀀스 학습에 부족. 모듈 단위 학습 등 추가 연구 필요.

🤖 8. 음악 생성

Algorithm 1: 규칙 기반

중첩행렬의 ON/OFF 패턴을 보고, "지금 켜진 패턴에 속한 음 중에서 랜덤으로 골라라".

  • 빠름 (2~3초)
  • 앞뒤 흐름 무시
  • 고아 note 보충 포함

Algorithm 2: 딥러닝

"이 패턴 조합일 때 원곡에서는 이 음을 쳤다"를 신경망이 학습.

  • FC: 매 순간 독립 판단. 가장 안정적
  • LSTM: 흐름을 기억하며 판단
  • Transformer: 곡 전체를 한눈에 보고 판단

onset 간격 제약

생성 음악이 화음 타건 연타처럼 들리는 문제 해결. min_onset_gap=3이면 1.5박마다 한 번 새 음.
